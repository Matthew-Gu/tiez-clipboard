## 目标

- 将设置初始化、显示、修改、保存和跨窗口同步迁移到统一设置业务层与 Zustand Store。
- 保持现有设置键、默认值和 Rust 状态同步行为不变。

## 执行顺序

- [ ] 迁移外观设置。
- [ ] 迁移通用设置。
- [ ] 迁移剪贴板设置。
- [ ] 迁移高级清洗策略。
- [ ] 迁移默认应用与数据管理设置。
- [ ] 收窄设置组件 props，禁止传入完整 AppState。
- [ ] 统一设置加载、解析、保存和错误处理。
- [ ] 保留 `settings-changed` 事件作为多窗口同步机制。
- [ ] 确认独立高级设置窗口复用同一设置服务。
- [ ] 删除已完成迁移的旧设置状态和同步分支。

## 兼容要求

- 不修改任何 `app.*` 设置键。
- 不修改字符串布尔值和数值解析语义。
- 不改变设置修改后的即时应用行为。
- 不假设 Zustand Store 可跨 Tauri 窗口共享。

## 验收场景

- 所有设置加载后与当前行为一致。
- 修改设置后 UI 立即更新，Rust 状态和数据库同步。
- 重启应用后设置保持。
- 独立高级设置窗口与主窗口通过事件同步。
- 设置保存失败不会错误覆盖已持久化状态。
