## 目标

- 拆分大型组件的展示、业务行为和窗口生命周期职责。
- 优先提取纯函数和无状态展示组件，保持现有 DOM 结构、样式类名和行为。

## 执行顺序

- [ ] 拆分前记录目标组件的明确复杂度问题和预期收益，无明确收益时延期。
- [ ] 仅提取被重复使用或能显著降低主体复杂度的富文本图片标记、表格识别和媒体预览纯函数。
- [ ] 为提取的预览函数补充测试。
- [ ] 将紧凑预览窗口管理从 ClipboardItem 展示组件中分离。
- [ ] 按内容类型拆分 ClipboardItem 的预览展示。
- [ ] 拆分 ClipboardItem 标签编辑与操作区。
- [ ] 拆分 TagManager 的数据行为、侧栏、条目区和对话框。
- [ ] 拆分复杂设置组中的重复控制结构。
- [ ] 保持现有 CSS 类名和可访问属性。
- [ ] 每次拆分仅做代码移动和依赖收窄，不同时改变业务行为。
- [ ] 不把简单函数拆成多个文件，不将只使用一次的逻辑抽成公共工具。
- [ ] 不创建超过现有业务域所需的额外目录层级。
- [ ] 不进行无关命名、格式化或风格统一。

## 高风险约束

- 不调整紧凑预览窗口尺寸、定位和生命周期时序。
- 不调整富文本降级策略。
- 不修改标签删除、重命名和敏感标签保护规则。
- 不改变固定项拖拽排序行为。
- 拆分后文件数量、目录层级和调用链增长必须有明确复杂度收益支撑。

## 验收场景

- 所有内容类型预览与拆分前一致。
- 主列表和紧凑预览显示一致。
- 标签管理增删改查和批量操作一致。
- 组件拆分后没有新增循环依赖或 `shared -> features` 反向依赖。
