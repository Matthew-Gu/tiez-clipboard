## 目标

- 集中管理前端使用的 Tauri 命令名、事件名、设置键和参数类型。
- 保持现有 Rust 命令、参数、返回数据与事件负载完全不变。

## 执行清单

- [ ] 统计重复、高频且容易写错的命令名、事件名和设置键，仅对这些契约建立常量。
- [ ] 仅在能够合并重复参数处理或错误处理时建立薄 service，不为每个命令机械创建 service。
- [ ] 为高频命令定义请求与返回类型。
- [ ] 优先迁移设置、历史分页和窗口导航调用。
- [ ] 按业务域逐步替换散落的字符串调用。
- [ ] 禁止 service 内引入业务状态或 UI 逻辑。
- [ ] 单次使用且参数清晰的 `invoke/listen` 默认保留原位。
- [ ] 不新增请求库或其他公共依赖。
- [ ] 补充关键 IPC 请求参数的契约测试。

## 兼容要求

- 不修改命令名、参数名、大小写和可选参数行为。
- 不修改事件名与事件 payload。
- 不修改错误字符串处理方式。
- 不在本阶段修改 Rust 业务实现。
- 新增契约模块必须缩短调用链或收敛至少两处真实重复，否则不实施。
